# Gameplay Notes

## Office Demo Mission

Играбельная демо-миссия `office` состоит из 4 шагов:

1. Подозрительное письмо от фальшивой ИТ-поддержки.
2. Проверка поддельного URL вместо перехода по ссылке.
3. Попытка социальной инженерии через запрос кода подтверждения.
4. Финальная реакция на инцидент: смена пароля, завершение сессий, эскалация в ИБ.

## Mechanics

- `Security HP` падает при опасных действиях и восстанавливается при безопасных.
- Каждый ответ фиксируется как `answer_event`, чтобы потом показывать статистику и типовые ошибки.
- После ошибки пользователь получает не просто пометку о неверном выборе, а текст последствий и подсказку.
- Две дополнительные ветки (`home`, `public-wifi`) заведены как модульные заготовки для масштабирования.

## Future Expansion

- Ветки для дипфейков, скимминга и безопасного использования API.
- Сертификат с QR-кодом и расширенная система лиг.
- Более насыщенная live-синхронизация через Redis Pub/Sub для многопользовательских сценариев.
